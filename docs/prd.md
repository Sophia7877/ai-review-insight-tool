# 产品需求文档：AI Review Insight Tool

## 1. 产品概述

AI Review Insight Tool 是一个面向品牌运营、电商运营、内容运营和产品运营场景的 AI 消费者评论洞察工具。

产品帮助运营人员将用户评论转化为可执行的业务洞察，包括购买动机、流失阻碍、高频痛点、内容选题、商品详情页优化建议和客服 FAQ。

线上 Demo：

```text
https://Sophia7877.github.io/ai-review-insight-tool/
```

## 2. 产品定位

本产品不是通用文本总结工具，而是一个面向运营工作流的 AI 分析工具。

它主要帮助团队回答五个问题：

- 用户为什么选择这个产品？
- 用户为什么犹豫或放弃？
- 哪些卖点应该被放大？
- 哪些疑虑应该在购买前被解释清楚？
- 如何把用户反馈转化为内容、详情页、客服和活动复盘动作？

## 3. 背景与问题

品牌和电商运营团队每天都会接触大量用户评论，但传统评论分析通常依赖人工阅读、手动归类和经验判断，效率低，也难以沉淀为可复用的运营动作。

| 问题 | 当前情况 | 本产品解决方式 |
| --- | --- | --- |
| 评论太多，看不过来 | 运营人员逐条阅读、复制重点评论 | 自动提取情绪、主题和代表性痛点 |
| 不知道用户为什么买 | 只看评分和销量，难以解释转化原因 | 从评论中识别购买动机和可放大卖点 |
| 不知道用户为什么不买 | 负面评论分散，难以归纳阻碍因素 | 提炼价格、成分、客服、物流等转化阻碍 |
| 内容选题靠感觉 | 内容方向依赖运营个人经验 | 从真实用户问题生成内容选题 |
| AI 输出不敢直接用 | 通用 AI 总结容易空泛 | 增加人工复核标准和可采纳指标 |
| 洞察难以复用 | 分析结果停留在报告里 | 输出周报、FAQ、详情页建议和结构化数据 |

## 4. 目标用户

### 品牌运营

需要从用户反馈中提炼产品卖点、传播角度、用户痛点和品牌沟通机会。

### 电商运营

需要优化商品标题、详情页、活动机制、客服话术和影响转化的关键问题。

### 内容运营

需要将真实用户问题转化为小红书、抖音、公众号、直播间和 FAQ 选题。

### 用户运营 / 客服运营

需要识别负反馈、复购机会、服务问题和高频用户疑虑。

## 5. 核心使用场景

### 场景 1：新品上市评论复盘

新品上线后，运营人员需要快速判断用户是否理解核心卖点，以及是否出现早期负反馈。

预期输出：

- 核心卖点感知情况
- 早期负反馈
- 详情页优化建议
- 上新后续内容方向

### 场景 2：大促活动复盘

大促或活动结束后，运营人员需要识别价格、赠品、物流、客服等影响转化和满意度的问题。

预期输出：

- 转化阻碍
- 活动规则理解问题
- 客服 FAQ
- 活动复盘总结

### 场景 3：内容选题规划

内容团队需要基于真实用户问题产出选题，而不是单纯依靠主观判断。

预期输出：

- 小红书选题
- 抖音短视频角度
- 直播间 FAQ
- 公众号文章方向

### 场景 4：商品详情页优化

电商团队需要把高频疑虑前置到详情页，在用户下单前降低决策成本。

预期输出：

- 详情页首屏应强化的信息
- 适用/不适用人群说明
- 成分或功能解释
- 价格和活动机制说明

## 6. 产品目标

### 业务目标

帮助运营团队把用户评论转化为可执行的运营动作，提高用户对产品价值的理解、信任和购买意愿。

### 用户目标

让运营人员可以通过复制评论、上传文件或加载样例，快速获得结构化评论洞察报告。

### 产品目标

建立一条轻量级工作流：

```text
评论输入 → AI 分析 → 人工复核 → 洞察采纳 → 运营动作 → 指标复盘
```

## 7. MVP 范围

### 7.1 评论输入

支持以下输入方式：

- 样例数据
- 文本框粘贴评论
- CSV/TXT 文件上传

### 7.2 评论分析

系统对评论进行分析并输出：

- 情绪分布
- 高频主题
- 代表性痛点
- 购买动机
- 流失阻碍

### 7.3 运营动作输出

将评论洞察转化为：

- 内容选题建议
- 商品详情页优化建议
- 客服 FAQ 建议
- 大促/活动复盘建议

### 7.4 导出与复用

用户可以：

- 复制周报总结
- 下载 Markdown 报告
- 下载结构化 JSON 数据

### 7.5 冷启动模板

Demo 提供三个早期激活模板：

- 新品上市复盘
- 大促活动复盘
- 差评与客服 FAQ 分析

## 8. 非 MVP 范围

当前版本暂不包含：

- 真实电商平台后台接入
- 自动抓取评论数据
- 实时数据看板
- 自动发布内容
- 替代人工业务判断
- 企业级权限管理

## 9. 用户旅程

```text
打开 Demo 页面
  ↓
选择运营场景模板
  ↓
加载样例评论或上传自己的评论
  ↓
生成洞察报告
  ↓
查看情绪、主题、痛点、内容选题和运营建议
  ↓
复制总结或下载报告/数据
  ↓
将洞察用于内容排期、详情页优化、客服 FAQ 或活动复盘
```

## 10. 核心功能

| 功能 | 说明 | 用户价值 |
| --- | --- | --- |
| 情绪识别 | 将评论分为正面、中性、负面 | 快速判断整体用户态度 |
| 主题识别 | 识别价格、效果、成分、包装、物流、客服等主题 | 找到用户最关心的问题 |
| 痛点提取 | 提取代表性负面评论 | 支持周报、复盘和优化计划 |
| 购买动机分析 | 判断用户为什么选择产品 | 放大有效卖点 |
| 流失阻碍分析 | 判断用户为什么犹豫或放弃 | 降低购买决策阻力 |
| 内容选题生成 | 将评论洞察转化为内容方向 | 支持内容运营和增长 |
| 报告导出 | 生成 Markdown 和 JSON 输出 | 让洞察可以复用 |
| 人工复核标准 | 定义采纳、修改、弃用标准 | 提高 AI 输出可信度 |

## 12. 早期用户获取策略

早期通过一个清晰、低门槛的使用钩子吸引第一批用户：

> 上传 30 条评论，10 分钟看清用户为什么买、为什么不买。

早期目标用户：

- 电商运营
- 内容运营
- 品牌运营
- 用户/客服运营

早期激活方式：

- 先展示样例报告，再要求用户输入数据
- 提供固定场景模板
- 支持复制粘贴评论
- 不要求接入平台后台
- 输出可复用的报告、内容选题、详情页建议和 FAQ 草稿

## 13. 风险与假设

### 假设

- 如果输出结果足够直接可用，运营人员愿意粘贴或上传评论数据。
- 小样本评论分析对早期产品验证有价值。
- AI 洞察进入业务流程前需要人工复核。

### 风险

- AI 输出可能过于泛化。
- 样例评论可能无法代表完整用户群体。
- 不同行业可能需要不同主题标签体系。
- 如果长期不接入真实平台数据，用户留存可能受限。
